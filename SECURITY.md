# Security policy

## Supported version

Security fixes target the current `main` branch. There are no separately
maintained release branches yet.

## Reporting a vulnerability

Please do not disclose a suspected vulnerability in a public issue. Use GitHub's
private vulnerability reporting feature from the repository's **Security** tab.
If private reporting is unavailable, contact the maintainer privately through
the GitHub account associated with the project before publishing details.

Include the affected URL or file, reproduction steps, impact, and any suggested
mitigation. Reports will be acknowledged as soon as practical, then assessed and
coordinated before public disclosure.
